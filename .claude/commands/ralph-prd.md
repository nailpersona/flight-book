# Ralph PRD - Створення Product Requirements Document

Створи PRD.md файл на основі отриманого завдання.

## Інструкція

1. **Уточни завдання**: Запитай користувача, для якої фічі/етапу потрібно написати PRD. Вивчай тільки ту частину проекту, яка стосується завдання. Якщо щось незрозуміло — задавай уточнюючі питання, поки все не стане ясним.

2. **Вивчай проект**: Після уточнення завдання проаналізуй відповідні файли, структуру та існуючий код. Не досліджуй проект без конкретного завдання.

3. **Створи детальний PRD.md** у форматі:
   ```markdown
   # Feature Name

   ## Overview
   Детальний опис фічі/етапу: мета, контекст, бізнес-вимоги.

   ## Technical Context
   Файли, таблиці БД, API, залежності — все, що стосується реалізації.

   ## Stories
   - [ ] Story 1: Детальний опис з критеріями приймання
   - [ ] Story 2: Детальний опис з критеріями приймання
   - [ ] Story 3: Детальний опис з критеріями приймання

   ## Notes
   Додаткові нотатки, ризики, залежності від інших задач.
   ```

4. **Вимоги до stories**:
   - Максимально деталізовані (що саме робити, які файли, які зміни)
   - Атомарні — одна story = одна чітка задача
   - Мають вимірюваний критерій виконання ( Done: "...")
   - Не залежать від інших незавершених stories

5. Після створення повідом про кількість stories та готовність до виконання.
